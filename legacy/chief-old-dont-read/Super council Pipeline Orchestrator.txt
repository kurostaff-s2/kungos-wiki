A2: Pipeline Orchestrator — Implementation                                                                                                                                             
                                                                                                                                                                                        
 ### Architecture: State Machine (Not Linear Pipeline)                                                                                                                                  
                                                                                                                                                                                        
 Why not linear? Council consensus: rigid pipelines break on failure. State machine allows adaptive recovery (Build fails → retreat to Scout → retry).                                  
                                                                                                                                                                                        
 ### Core Class: PipelineState                                                                                                                                                          
                                                                                                                                                                                        
 ```python                                                                                                                                                                              
   # Lives alongside ChairGateState in super-council.py                                                                                                                                 
   # Serialized to ~/.council-memory/pipelines/{pipeline_id}.json                                                                                                                       
                                                                                                                                                                                        
   VALID_TRANSITIONS = {                                                                                                                                                                
       SCOUT    → [PLAN, FAILED]                                                                                                                                                        
       PLAN     → [BUILD, SCOUT, FAILED]    # retreat to scout if plan blocked                                                                                                          
       BUILD    → [VALIDATE, SCOUT, FAILED]  # scout on failure                                                                                                                         
       VALIDATE → [GATE, BUILD, FAILED]      # retry build                                                                                                                              
       GATE     → [INDEX, BUILD, FAILED]     # retry on gate fail                                                                                                                       
       INDEX    → [DONE, FAILED]                                                                                                                                                        
       DONE     → []                          # terminal                                                                                                                                
       FAILED   → [SCOUT, DONE]               # reprobe or abort                                                                                                                        
   }                                                                                                                                                                                    
 ```                                                                                                                                                                                    
                                                                                                                                                                                        
 State persisted per phase:                                                                                                                                                             
 - pipeline_id (UUID, idempotent key)                                                                                                                                                   
 - phase (current phase)                                                                                                                                                                
 - history (list of {phase, status, duration_ms, error})                                                                                                                                
 - artifacts (phase → output path mapping)                                                                                                                                              
 - retry_count (max 3)                                                                                                                                                                  
                                                                                                                                                                                        
 ### Endpoint: POST /v1/council/pipeline                                                                                                                                                
                                                                                                                                                                                        
 ```                                                                                                                                                                                    
   Payload: {task, pipeline_id?, phase?}                                                                                                                                                
 ```                                                                                                                                                                                    
                                                                                                                                                                                        
 Flow:                                                                                                                                                                                  
 1. Load or create PipelineState (idempotent via pipeline_id)                                                                                                                           
 2. Execute current phase via _execute_phase()                                                                                                                                          
 3. Record outcome in history                                                                                                                                                           
 4. Auto-advance or adaptive recovery:                                                                                                                                                  
     - Success → advance to next phase                                                                                                                                                  
     - Failure + retries left → retreat to Scout                                                                                                                                        
     - Failure + max retries → FAILED state

 ### Core Methods                                                                                                                                                                       
                                                                                                                                                                                        
 _verify_spec_backbone(task_id) — checks all 3 files exist AND contain required sections:                                                                                               
                                                                                                                                                                                        
 ```python                                                                                                                                                                              
   Returns: {pass, missing, artifacts}                                                                                                                                                  
   - pass: True if all artifacts complete                                                                                                                                               
   - missing: list of missing artifacts/sections                                                                                                                                        
   - artifacts: dict of artifact → path                                                                                                                                                 
 ```                                                                                                                                                                                    
                                                                                                                                                                                        
 _scaffold_spec(task_id, task) — generates empty templates:                                                                                                                             
                                                                                                                                                                                        
 ```python                                                                                                                                                                              
   Returns: {paths, status: "scaffolded"}                                                                                                                                               
   - Creates all 3 files with template content                                                                                                                                          
   - Idempotent (overwrites if exists)                                                                                                                                                  
 ```

 ### Hard Gate Integration                                                                                                                                                              
                                                                                                                                                                                        
 _pre_build_gate(task_id) — called in _delegate() before model swap:                                                                                                                    
                                                                                                                                                                                        
 ```python                                                                                                                                                                              
   If spec incomplete → 412 Precondition Failed                                                                                                                                         
     Error: "Spec backbone incomplete. Missing: [...]"                                                                                                                                  
     Action: "Run /v1/council/spec/scaffold to generate templates"                                                                                                                      
                                                                                                                                                                                        
   If spec complete → pass                                                                                                                                                              
 ```

 Combined Flow (A2 + D4)                                                                                                                                                                
                                                                                                                                                                                        
 ```                                                                                                                                                                                    
   User: "Implement auth module"                                                                                                                                                        
        ↓                                                                                                                                                                               
   POST /v1/council/pipeline {task: "Implement auth module"}                                                                                                                            
        ↓                                                                                                                                                                               
   Phase: SCOUT                                                                                                                                                                         
     → _active_recall("auth module")                                                                                                                                                    
     → Returns: existing auth patterns, related code                                                                                                                                    
        ↓                                                                                                                                                                               
   Phase: PLAN                                                                                                                                                                          
     → _scaffold_spec(task_id, task)  ← D4 integration                                                                                                                                  
     → _delegate(planner, context=scout_results)                                                                                                                                        
     → Planner fills in all 3 artifacts                                                                                                                                                 
     → _verify_spec_backbone(task_id) ← D4 gate                                                                                                                                         
     → If incomplete → retreat to SCOUT                                                                                                                                                 
        ↓                                                                                                                                                                               
   Phase: BUILD                                                                                                                                                                         
     → _pre_build_gate(task_id) ← D4 hard gate                                                                                                                                          
     → If incomplete → 412 Precondition Failed                                                                                                                                          
     → _delegate(builder, spec=artifacts)                                                                                                                                               
        ↓                                                                                                                                                                               
   Phase: VALIDATE                                                                                                                                                                      
     → _chair_gate()                                                                                                                                                                    
        ↓                                                                                                                                                                               
   Phase: GATE                                                                                                                                                                          
     → POST /v1/council/chair-gate                                                                                                                                                      
        ↓                                                                                                                                                                               
   Phase: INDEX                                                                                                                                                                         
     → _auto_index_file(build_output)                                                                                                                                                   
        ↓                                                                                                                                                                               
   Phase: DONE                                                                                                                                                                          
 ```                                                                                                                                                                                    
                                                                                                                                                                                        
 Key integration point: The spec gate (D4) fires at two points:                                                                                                                         
 1. During PLAN — verifies artifacts are complete before advancing to BUILD                                                                                                             
 2. Before BUILD — hard gate blocks model swap if spec incomplete                                                                                                                       
                                                                                                                                                                                        
 This ensures the system cannot proceed without a plan, per council consensus.
