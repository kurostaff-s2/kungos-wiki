# Task 04: Inbound Sync with Revision Awareness

**Source spec:** `~/.llm-wiki/super-council-docs/12-appflowy-integration-v2.1.md` §7.2, §7.3, §10.1
**Generated:** 01-06-2026
**Goal:** Poll AppFlowy for user edits, enforce field allowlists, compute SHA-256 snapshot hashes, and route changes through Council Core API with revision checks.

---

## Scope

Background worker that detects changes in AppFlowy rows, compares against persisted snapshot hashes, and applies approved edits to Council.

### Steps

1. **Snapshot hash:** Implement `compute_snapshot_hash(cells: dict) -> str` as SHA-256 over canonical JSON:
   - Object keys sorted lexicographically
   - No insignificant whitespace (`separators=(',', ':')`)
   - Stable serialization for arrays in original AppFlowy-provided order
   - Nulls preserved
   - UTF-8 encoding before hashing
2. **Polling:** Call `GET /api/workspace/{ws}/database/{db}/row/updated` to detect recently changed rows.
3. **Fetch detail:** For each changed row, call `GET /api/workspace/{ws}/database/{db}/row/detail` to get full cell payload.
4. **Resolve binding:** Lookup `appflowy_bindings` by `(appflowy_database_id, appflowy_row_id)`.
5. **Hash comparison:** Canonicalize cells → compute SHA-256 → compare with `binding.last_seen_cells_hash` → skip if identical.
6. **Field allowlist:** For each changed field, check against `FIELD_ALLOWLIST[entity_type]`. Forbidden fields → INSERT `inbound_changes` with `state='rejected'`, `rejection_reason='forbidden_field'`.
7. **Mapping transform:** Convert AppFlowy cell values to Council field values (e.g., AppFlowy select → enum string).
8. **Revision-aware PATCH:** Fetch canonical entity from Council Core API → extract current revision → `PATCH /v1/<resource>/{id}` with `expected_revision=current`. On `409` → INSERT `inbound_changes` with `state='rejected'`, `rejection_reason='stale_revision'`.
9. **Record result:** INSERT `inbound_changes` with `state='applied'` for successful edits.
10. **Persist hash:** Update `binding.last_seen_cells_hash` AFTER successful processing (not before).

```python
FIELD_ALLOWLIST = {

    "review": {"notes"},
    "review_finding": {"finding_state", "owner_note"},
    "prompt_template": {"name", "template_body", "tags", "status"},
    "knowledge_card": {"title", "body", "tags"},
}
```

---

### Constraints

- Hash must be deterministic: same input → same hash, every time.
- `last_seen_cells_hash` read from DB on worker start (NOT process memory).
- Hash persisted AFTER processing (not before) to avoid losing change detection on crash.
- Forbidden fields logged as `rejected`, not silently dropped.
- Stale revision → `rejected` with reason, not retry loop.
- One inbound change record per field per poll cycle.

---

### Success Criteria

- [ ] Snapshot hash is deterministic (same input → same hash)
- [ ] Snapshot hash is sensitive (one cell change → different hash)
- [ ] Unchanged rows skipped (hash match)
- [ ] Approved fields applied through Council Core API
- [ ] Forbidden fields rejected and logged
- [ ] Stale revision detected and logged
- [ ] `last_seen_cells_hash` persisted in DB (restart-safe)
- [ ] Worker restart does not lose or duplicate change detection

---

### Tests

1. **Hash determinism:** `compute_snapshot_hash({"a": 1, "b": 2})` called 100 times → assert all identical.
2. **Hash sensitivity:** Change one value → assert different hash.
3. **Hash canonicalization:** `{"b": 2, "a": 1}` and `{"a": 1, "b": 2}` → assert same hash (key sorting).
4. **Allowlist enforcement:** Edit `work_items.kind` in AppFlowy → assert `inbound_changes` row with `state='rejected'`, `rejection_reason='forbidden_field'`.
5. **Approved field applied:** Edit `work_items.title` in AppFlowy → assert `inbound_changes` row with `state='applied'` + Council entity updated.
6. **Stale revision:** Edit in AppFlowy after Council-side edit → assert `rejected` with `stale_revision`.
7. **Restart safety:** Process 5 changes → kill worker → restart → assert no duplicate processing (hash persisted).
8. **Null handling:** Cell with null value → preserved in hash (not omitted).
